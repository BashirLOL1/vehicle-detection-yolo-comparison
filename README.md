«Детекции транспортных средств для дорожных камер»

Тасуев Башир Ахмедович
Группа: БВТ2501

 
Введение
Актуальность темы: С ростом интенсивности дорожного движения возрастает потребность в автоматизированных системах контроля и анализа транспортных потоков. Использование систем компьютерного зрения на дорожных камерах позволяет в реальном времени обнаруживать транспортные средства, оценивать плотность потока и выявлять нарушения. Это снижает нагрузку на операторов видеонаблюдения и повышает безопасность дорожного движения.
Предметная область: интеллектуальные транспортные системы, автоматизированный анализ видеопотока, детекция объектов на изображениях.
Цель работы: обучить и сравнить пять современных архитектур нейронных сетей для детекции транспортных средств (легковые автомобили, грузовики, автобусы, мотоциклы) на дорожных камерах, чтобы определить оптимальную модель для реального времени.
Задачи работы:
1.	Провести обзор литературы для анализа существующих подходов к детекции объектов.
2.	Подготовить размеченный датасет для обучения моделей.
3.	Реализовать и обучить пять архитектур нейронных сетей (YOLOv8n, YOLOv10n, YOLOv11n, YOLO26n, YOLOv11s).
4.	Выполнить экспериментальную оценку качества моделей с использованием метрик mAP50, Precision и Recall.
5.	Провести сравнительный анализ результатов и сформулировать выводы о применимости рассмотренных методов.


Постановка задачи
Цель задачи: классическая задача компьютерного зрения — детекция объектов. Необходимо выделить транспортные средства на изображении «рамкой» (bounding box) и классифицировать их тип (car, motorbike, bus, truck).
Входные данные: изображения с дорожных камер 
Выход модели: координаты рамок вокруг обнаруженных транспортных средств и вероятность класса.
Метрики оценки:
•	mAP50 (mean Average Precision при IoU=0.5) — основная метрика точности детекции.
•	Precision (точность) — доля верных срабатываний.
•	Recall (полнота) — доля найденных объектов от всех реальных.
•	Время обучения — практический показатель для внедрения.


Методы и модели
Как устроены модели:
1.	Одностадийные модели (YOLOv8n, YOLOv10n, YOLOv11n, YOLO26n, YOLOv11s): Эти модели не тратят время на раздумья. Когда фото попадает внутрь, сеть сразу выдаёт готовый список объектов.
o	YOLOv8n — классическая лёгкая модель, проверенный базовый вариант.
o	YOLOv10n — улучшенная версия с оптимизациями, быстрее предшественника.
o	YOLOv11n — современная модель с высоким балансом скорости и точности.
o	YOLO26n — новейшая модель 2026 года, не требующая постобработки NMS.
o	YOLOv11s — более глубокая версия YOLOv11 с увеличенным числом параметров для лучшей точности.
Обоснование выбора: Выбраны модели семейства YOLO как наиболее популярные для задач реального времени. Разные версии (nano и small) позволяют оценить влияние размера модели на качество детекции.
Параметры обучения (гиперпараметры):
•	Batch = 16 (для nano-версий) / 12 (для small-версий)
•	Device = 0 (GPU) — NVIDIA RTX 3060
•	ImgSz = 640 — стандартный размер входного изображения
•	Workers = 4 — потоки для загрузки данных
•	Epochs = 50 — достаточное количество для сходимости
•	AMP = True — смешанная точность для ускорения

Выводы по результатам сравнения
1.	Лучшая по точности: YOLOv11s показала наивысший результат mAP50 = 91.43%, что делает её лучшим выбором для задач, где критична максимальная точность обнаружения.
2.	Самая лёгкая и быстрая: YOLOv10n (2.3M параметров, 7 часов обучения) показывает отличный баланс между размером модели и точностью (90.12%), что делает её предпочтительным выбором для внедрения на граничных устройствах с ограниченными ресурсами.
3.	Новейшая технология: YOLO26n (2026 год) достигла точности 90.20% при меньшем времени обучения (7 часов) и не требует постобработки NMS, что упрощает инференс.
4.	Аномалия YOLOv11n: Модель YOLOv11n показала аномально низкие результаты (76.18%), что предположительно связано с некорректным завершением обучения (перезагрузка компьютера).
5. Сравнение моделей показало, что увеличение размера модели (YOLOv11s vs YOLOv11n) даёт существенный прирост точности (+15%), но требует больше вычислительных ресурсов. Наиболее сбалансированной для внедрения на ограниченном оборудовании является YOLOv10n.
